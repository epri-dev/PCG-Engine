PCG Documentation¶

Welcome! This is the documentation for PCG Builder version 1., last updated Jun 25, 2020.

What is PCG Builder?

PCG Builder is a Python package that provides methods for constructing protection control groups (PCG) from node-breaker powerflow models.

License

PCG Builder is released under a BSD 3-clause license.

Contents

1. Getting Started¶

This section describes how to get started with PCG Builder. In particular, it covers installation and provides a quick example that shows how to use this package.

1.1. Installation¶

In order to install PCG Builder, the following tools are needed:

  • Linux and macOS: not currently supported

  • Windows: Python (2 or 3)

After confirming these pre-requisites are met, confirm that ‘pip’ and ‘py’ can be recognized as valid commands from your command prompt and that the default python release on your computer is the one you wish to install PCG Builder with. This can be easily tested by executing the following commands in the command prompt:

pip

py
exit()

Next the Python dependencies can be easily installed by executing the following command:

pip install pythonnet

Alternatively, the specific version of python where the package should be installed can be specified by executing the following instead:

For python 3:
py -3 -m pip install pythonnet

For python 2:
py -2 -m pip install pythonnet

To install the PCG Python module from source, first obtain the code from EPRI’s GitHub Repository, and then execute the following commands in command prompt from the root directory of the downloaded package:

python setup.py install

To build and install from a wheel file, first obtain the code from EPRI’s GitHub Repository, and then execute the following commands in command prompt from the root directory of the downloaded package:

python setup.py bdist_wheel
pip install --no_index pcg_builder

A more user friendly option may be to install the module from the pre-compiled wheel file. To do this, obtain the code from EPRI’s GitHub Repository, and then execute the following commands in command prompt from the root directory of the downloaded package:

pip install --no_index pcg_builder

1.2. Example¶

The following example shows how to retrieve buses and branches associated with PCGs using PCG Builder:

>>> import epri.pcg_builder as pcg_builder

>>> # Parse powerflow file
>>> parser = pcg_builder.parser.Parser()
>>> print(parser.parse_file('sample_case.raw'))

>>> # Build Protection Control Groups
>>> parser.build_pcgs(debug=True)

>>> # Get number of PCGs
>>> n_pcgs = parser.get_num_pcgs()

>>> # Print basic PCG summary in console
>>> for indx in range(n_pcgs):
      pcg = pcg_builder.pcg.PCG(parser.file_parser, indx)
      pcg_branches = pcg.get_branches()
      pcg_buses = pcg.get_pcg_buses()
      print('PCG #{}:'.format(indx))
      print('\tBuses:')
      for bus_number, bus_data in pcg_buses.items():
              print(bus_number, bus_data)
      print('\tBranches:')
      for branch_data in pcg_branches:
              print(branch_data)

In this example, it is assumed that the Python interpreter is started from a directory where sample-case is located.

2. Parser¶

The PCG Builder library contains read-only file parsers for two common powerflow model raw data file formats:

  • PTI PSSE: version 34

  • GE PSLF: version 19+

After parsing the raw model data from the supplied powerflow model file, network related data is converted to a common model that contains mainly connectivity information. Other data, including element type and impedance, relevant for constructing contingencies applicable to PCGs is also retained.

The basis of the common model is the substation which consists of nodes (bus segments), breakers that isolate bus segments, branches and transformers that connect nodes, and shunt elements connected to nodes.

3. PCG¶

Protection control groups (PCGs) are sections of the power grid comprised of a collection of branches, bus sections, shunt elements, and generators which are isolated from faults on the network by coordinated circuit breaker operations that take all the contained elements out of service. PCG Builder constructs these PCGs through a graph theory based approach which groups together network nodes connected by network branches until all nodes inside the PCG are only connected one another and to nodes outside the PCG via circuit breakers. The defined PCG also inherits any shunt elements, or generators connected to included nodes as well as branch connections between the nodes. Circuit breakers are commonly shared by two adjacent PCGs.

3.1. Study Area¶

Oftentimes the entire powerflow network is not of interest and only a subset of the PCGs are desired. Identifying this subset is handled by defining a study area. The study area is comprised of complementary sets of Areas, Zones, Owners, Substations, Buses, and KV Ranges identifiers. Where multiple subset identification fields are supplied, the resulting study area will be comprised of the intersection of all the supplied identifiers. Identification fields left blank are not considered in constructing the study subsystem.

3.1.1. Areas¶

Areas are an identification field commonly used in powerflow models to allow network elements to be easily grouped together. Though the application of area labels is arbitrary, convention typically dictates that area designations are the most encompassing field. Usually an area includes an entire geographic region, balancing authority, reliability coordination region, or other high level administrative membership.

See: epri.pcg_builder.parser.Parser.set_study_areas()

3.1.2. Zones¶

Though the application of zone labels is also arbitrary, convention typically dictates that zone designations are specific to a local region. Usually a zone includes an specific load center, maintenance region, or other localized control membership.

See: epri.pcg_builder.parser.Parser.set_study_zones()

3.1.3. Owners¶

Owner labels typically denote real world equipment ownership.

See: epri.pcg_builder.parser.Parser.set_study_owners()

3.1.4. Substations¶

Substation labels denote real world substation membership.

See: epri.pcg_builder.parser.Parser.set_study_substations()

3.1.5. Buses¶

See: epri.pcg_builder.parser.Parser.set_study_buses()
